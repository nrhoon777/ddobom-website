# 1. 기획 (Service Planning)

## 1.1 목표 / KPI
4대 목표 중 **후원 전환·신뢰**가 사업적으로 가장 중요.

| 영역 | 지표 | 목표(예시) |
|---|---|---|
| 후원(핵심) | 온라인 후원 전환율, 정기후원 신규/유지, 결제 성공률 | 정기후원 6개월 +30%, 성공률 98%+ |
| 후원 신뢰 | 기부금영수증 자동발급률, 사용내역·연차보고 도달률 | 영수증 자동발급 100% |
| 접근성 | KWCAG 2.2 준수율 / WA 인증 | 자동검사 오류 0건 |
| 정보전달 | 사업·캠페인 체류시간, 검색 성공률 | 체류시간 +20% |
| 커뮤니티·참여 | 프로그램 신청 수, 환우 커뮤니티 활성, 봉사 신청 | 프로그램 신청 +40% |
| 반응형 | 데스크톱 이탈률 | -20% |

## 1.2 페르소나
- **A. 당사자 윤지우(29, 치료 중)** — 익명으로 또래와 연결, 프로그램 참여. → 익명 닉네임 활동, 부담 없는 신청, 따뜻한 톤, 큰 글씨
- **B. 후원자 한도윤(38, 직장인)** — 신뢰 단체에 정기후원, 영수증 필요. → 3분 내 간편/정기결제, 재정 투명성, 영수증 자동발급
- **C. 기업 CSR 정세림(41)** — 협력·물품 후원 검토. → 기업/물품 후원 동선, 성과·연차보고, 담당 연결
- **(보조) D. 자원봉사자/언론 임하준(24)** — 캠페인 동참·재능기부·취재. → 봉사 신청, 보도자료

## 1.3 핵심 시나리오
1. **정기 후원(최우선)** — 홈 → 후원하기 → 유형·금액 → 정보·영수증 동의 → PG 결제 → 완료·자동 영수증 → 마이페이지 내역
2. **프로그램 참여** — 사업·활동 → 프로그램 상세 → 신청(익명/로그인) → 동의·입력 → 접수
3. **환우 커뮤니티** — 닉네임 가입 → 커뮤니티 → 익명 글·댓글 → 위로·정보 교환
4. **신뢰 확인 후 후원** — 소개·대표 이야기 → 성과 → 사용내역·연차보고 → 후원/기업제휴
5. **자원봉사·재능기부** — 참여하기 → 모집 상세 → 신청
6. **운영(관리자)** — CMS → 후원·영수증/캠페인·게시판/신청자/사용내역·연차보고 관리

## 1.4 정보구조(IA) / 사이트맵

# 2. 디자인 (UX/UI)

## 2.1 컨셉 — "다시, 봄처럼"
키워드 **청년(20·30) · 암 경험 · 희망(또봄) · 따뜻함 · 신뢰**. 무드는 **Warm & Hopeful, but Grounded**.
- "환자"가 아닌 "다시 살아가는 청년" — 동정(pity)이 아닌 **연대·응원**의 톤
- **금지:** 병상·수액·삭발 등 투병 클리셰, 공포·슬픔 자극, 미동의 환자 신원 노출
- 사진/일러스트: 회복·일상·관계(여행·모임·웃음), 신원 비식별(익명·실루엣·동의 사진), 봄·새싹·빛 모티프

## 2.2 반응형 — 모바일 우선
SNS(또또봄TV·인스타) → 모바일 랜딩 유입이 크고, **후원 결제의 모바일 완결성이 전환율의 핵심**. 데스크톱은 임팩트 스토리·재정 투명성 정독형 위계로 재설계.

| 구분 | 범위 | 컬럼 |
|---|---|---|
| Mobile | ~767px | 4 (햄버거 GNB, 후원 CTA 스티키) |
| Tablet | 768~1023px | 8 |
| Desktop | 1024~1439px | 12 |
| Wide | 1440px~ | 12 (컨테이너 max 1200~1280) |

## 2.3 디자인 시스템 (실제 코드 토큰 반영)
- **컬러:** Primary `#1f9d6b`(새싹 그린·재생), Secondary `#ff7a59`(따뜻한 코랄), Accent `#ffc94d`(봄 햇살), Blossom `#ffd9e0`. 그린/코랄 위 텍스트 대비 별도 점검.
- **타이포:** Pretendard 가변폰트, 본문 16px+, line-height 1.7, rem 기반. (선택)캠페인 타이틀용 악센트 폰트
- **컴포넌트:** 버튼(Primary/Secondary/Ghost/**Donate 강조**), 폼(+금액 칩·정기/일시 토글·결제수단), 카드, GNB·햄버거·스티키 후원바, 표→카드, 탭/아코디언, **임팩트 컴포넌트(통계 카운터·진행률 게이지·재정 투명성 카드·인증 배지)**, 영상 임베드

## 2.4 KWCAG — 디자인 책임
색대비 4.5:1 / 색 비의존 / 가시적 포커스(결제 단계 특히) / 터치 44px+ / alt 정책(신원 비식별) / 키보드 / 200% 무손실 / 모션 축소 / 폼·결제 오류 안내 / 평이·존중 어조

## 2.5 주요 레이아웃 & 반응형 변환
- **후원 랜딩(신규·핵심):** 임팩트 히어로 → 정기/일시 토글+금액 칩 → 임팩트 환산("○○원=○○ 지원") → 진행률 게이지 → 후원자 스토리 → 재정 투명성·영수증 → 결제. 데스크톱 좌설명/우폼 2단 → 모바일 1단 + 하단 스티키 결제 CTA
- **임팩트 스토리(신규):** 스토리 카드 → 상세(신원 비식별·인용 강조)
- 공통: GNB→햄버거, 표→카드, 필터→바텀시트, 후원 CTA 모바일 스티키 상시 노출

# 3. 개발 (Engineering)

## 3.1 기술 스택 & CMS
**권장:** Next.js(반응형·SEO·접근성 통제) + **후원·회원 전용 백엔드(NestJS)** + PostgreSQL + Redis + 오브젝트 스토리지 + CDN/WAF.

**CMS:** 헤드리스 **Strapi** 1순위(콘텐츠 자유·RBAC·민감 도메인 분리) / 헤드리스 **WordPress** 절충(편집 UX·인력). 그누보드는 비권장.
> **핵심 원칙: 결제·회원·건강 민감정보는 CMS가 아닌 전용 백엔드에서 처리** — 어느 CMS든 이 분리 유지.

## 3.2 아키텍처
```
[브라우저] ─HTTPS→ [CDN+WAF] → [Next.js(SSR/SSG/ISR)]
   ├─→ [CMS API(Strapi)] → [PostgreSQL(콘텐츠)] + [오브젝트 스토리지]
   └─→ [후원·회원 백엔드(NestJS)] → [PostgreSQL(회원/후원/거래)] + [Redis]
              ├─→ [PG 빌링/결제 API] (정기/일시)
              ├─→ [기부금영수증/국세청 연계(검토)]
              └─→ [알림(메일/문자) · 유튜브(또또봄TV)]
```
- 정기결제 빌링키·카드정보는 **PG 토큰화 위임**(자체 미보관, PCI 범위 최소화)
- 후원/회원/건강정보 DB는 콘텐츠 DB와 분리. 국내 리전 권장
- 도메인: stage 검수 → 컷오버 시 DNS 전환, 301 매핑, HTTPS+HSTS

## 3.3 반응형 / 접근성 구현
모바일 퍼스트·유동형, CSS Grid/Flexbox + `rem`/`clamp()` + 디자인 토큰 변수화, 컴포넌트 라이브러리(Storybook), 유튜브 facade. 시맨틱 마크업·ARIA·키보드·CI 접근성 게이트(axe/pa11y/Lighthouse) + 스크린리더 수동점검(결제 경로 특히).

## 3.4 회원·커뮤니티·후원
- **RBAC(센터 격리 폐기, 단일 법인):** 사무국(콘텐츠/후원회계/최고) · 환우 · 후원자 · 자원봉사자 · 비회원. **역할은 다대다 속성**(환우이자 후원자 가능)
- 익명 닉네임 가입과 후원자(실명·결제) 정보 **분리 저장**
- 커뮤니티: 비공개·회원전용·익명 옵션, 신고·모더레이션·위기 키워드 감지
- **후원(1순위):** 일시 + 정기(빌링), PG 비교(토스페이먼츠/나이스페이/KG이니시스), 멱등성·웹훅 검증·실패 상태머신, 후원 대시보드, 사용내역/연차보고 CMS

## 3.5 마이그레이션 / 보안
- 인벤토리 → 매핑 → 회원·건강정보(최소수집 재검토) → **후원 이력(세무 보존)·정기결제 빌링키는 PG 종속→재등록 고지** → ETL·스테이징 리허설 → 301·롤백
- **보안:** 건강=민감정보(별도 동의·암호화·익명 옵션), 결제 PCI 최소화·전자금융 규정·2FA·감사로그, OWASP·WAF·CSP

## 3.6 일정(WBS) 개략
착수 2–3주 / 설계 2–3주 / 인프라 2–3주 / 프론트·컴포넌트 3–4주 / 회원·커뮤니티 3–4주 / **후원 시스템(PG·영수증·대시보드) 4–5주** / 마이그레이션 2–3주 / 접근성·QA·보안 2–3주 / 인증·컷오버 2주+ → **약 5.5~7개월** (후원·결제로 +1개월)
